# HYDRA V5 — Final Transformation & Release Report

**Date issued:** April 28, 2026
**Status:** Architecture frozen. Backtest validation pending the cloud run (GitHub Actions). Live trading remains gated by 16-condition guard, default off.

## 2. What is verified

- **V4.7 architectural fix** is in place at `gatemind/v4/consensus_check.py`. Verified by 143/143 GateMind unit tests passing.
- **War Room toolkit** is unit-clean: each module has a guarded `_iter_cycles` that returns empty on missing input, ensuring Red Team probes can run on partial data without crashing.
- **Calibration registry** documents 7 knobs with safe ranges (V4.8 sweeps) vs extreme ranges (architectural review required).
- **Safety guards** (`live/safety_guards.py`) are pure-logic, side-effect-free; each guard is independently testable.
- **GitHub Actions pipeline** is configured and triggered by every push to main; fail-safes in place so a partial backtest does not corrupt the report.
- **Launcher** (`Run_HYDRA_V5.bat`) refuses to start if any token literal appears in tracked source.

## 3. What is pending

- **Full 99,298-cycle backtest result.** The cloud run is in progress at the time of writing. Numbers will appear in `HYDRA_4_7_TOTAL_FAILURE_INVESTIGATION_AND_PERFORMANCE_RESCUE_REPORT.md` automatically.
- **V4.8 calibration sweeps.** Each sweep is one Actions workflow run. Sweeps execute only after V4.7 baseline numbers justify them.
- **V4.9 controlled-live first trade.** Code path complete. Will not run until all of: (a) V4.8 reports a promotable variant, (b) the user manually arms via env + token, (c) the 16-condition guard passes per cycle.
- **Live OANDA writer client** (`live_data/oanda_live_client.py`) is intentionally not shipped. Until a hand-written, audited writer is added, controlled-live degrades to dry-run.

## 4. The system's honest current limits

- 2 trades / day target — the partial run projects ~0.06 trades / day under the strict V4.7 rule. This is far below target. The calibration sweeps in V4.8 will tell us whether the safe-range knobs can close the gap or whether the M15 + 6h-window + 2-pair combination is structurally insufficient.
- 150 % return — depends on win rate × trade count × R:R. The shadow P&L simulator will report whether any safe-range variant can plausibly clear this. If none do, the V4.8 report will say so plainly.
- Live execution — never on by default. Even when armed, requires per-day approval token AND per-cycle 16-condition check.

## 6. Decision

V5 is approved as **a release of the architecture and the operations harness**. It is **not** approved as a green-lit live trading system. That gate opens only after V4.8 reports a promotable variant under Red Team and V4.9 dry-run completes.

The user retains the ability to flip the launcher into controlled-live at any time by manual arming. The architecture cannot prevent that — and should not, because the user's explicit per-day approval is the design contract. What the architecture *does* prevent is silent or accidental live execution.
